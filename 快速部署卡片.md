## 🔑 环境变量速查表

### 必需的 8 个变量

```bash
DATABASE_URL=postgresql://...           # 从 Neon 复制
OPENAI_API_KEY=sk-...                  # 从 DeepSeek 复制
OPENAI_BASE_URL=https://api.deepseek.com/v1
OPENAI_MODEL=deepseek-chat
SECRET_KEY=...                         # 从 randomkeygen.com 生成
CORS_ORIGINS=https://...vercel.app     # 你的 Vercel URL
DEBUG=False
WEWE_RSS_URL=http://localhost:4000
```

### 可选的 1 个变量

```bash
REDIS_URL=https://...                  # 从 Upstash 复制（可选）
```

## ⚡ 30分钟部署流程

### 第一阶段：准备环境变量（15分钟）

1. **Neon 数据库**（3分钟）
   - 访问 https://neon.tech/
   - GitHub 登录 → Create project
   - 项目名：`wechat-articles`
   - 区域：Singapore
   - 复制 Connection string

2. **DeepSeek AI**（5分钟）
   - 访问 https://platform.deepseek.com/
   - 手机号注册 → 充值 10元
   - API Keys → 创建新密钥
   - 复制 API Key

3. **SECRET_KEY**（1分钟）
   - 访问 https://randomkeygen.com/
   - 复制 "Fort Knox Passwords" 中的一个

4. **Vercel URL**（1分钟）
   - 打开 Vercel Dashboard
   - 复制你的项目 URL

5. **Upstash Redis**（可选，3分钟）
   - 访问 https://upstash.com/
   - GitHub 登录 → Create Database
   - 复制 REST URL

6. **填写模板**（2分钟）
   - 打开 `环境变量模板.txt`
   - 填入所有值
   - 保存备用

---

### 第二阶段：部署后端（10分钟）

1. **创建 Render 服务**（3分钟）
   - 访问 https://render.com/
   - GitHub 登录
   - New + → Web Service
   - 选择 `j-lab-10404/wechat-articles`
   - 配置：
     ```
     Name: wechat-articles-backend
     Region: Singapore
     Branch: main
     Root Directory: backend
     Runtime: Docker
     Dockerfile Path: docker/backend.Dockerfile
     Instance Type: Free
     ```
     （注意：Dockerfile Path 是 `docker/backend.Dockerfile`，不需要 `../`）

2. **添加环境变量**（5分钟）
   - 在 Environment Variables 部分
   - 逐个添加你准备好的 8 个变量
   - 点击 Create Web Service

3. **等待部署**（5-10分钟）
   - 等待状态变为 "Live"
   - 复制后端 URL

---

### 第三阶段：连接前后端（5分钟）

1. **初始化数据库**（2分钟）
   - Render Dashboard → Shell
   - 运行：
     ```bash
     python -c "from app.database import init_db; init_db()"
     ```

2. **配置前端**（3分钟）
   - Vercel Dashboard → Settings → Environment Variables
   - 添加：
     ```
     Key: VITE_API_BASE_URL
     Value: https://你的后端URL/api
     ```
   - Deployments → Redeploy

---

### 第四阶段：测试（5分钟）

1. **测试后端**
   - 访问：`https://你的后端URL/docs`
   - 应该看到 API 文档

2. **测试前端**
   - 访问：`https://你的前端URL`
   - 应该能正常打开

## 🔧 故障排查

### 后端无法访问？

```bash
# 检查 Render 日志
Render Dashboard → Logs

# 检查环境变量
Render Dashboard → Environment → 确认所有变量都已设置

# 重新部署
Render Dashboard → Manual Deploy → Deploy latest commit
```

### 前端无法连接后端？

```bash
# 检查 Vercel 环境变量
Vercel Dashboard → Settings → Environment Variables
确认 VITE_API_BASE_URL 正确

# 检查后端 CORS
Render Dashboard → Environment → CORS_ORIGINS
确认包含前端 URL

# 重新部署前端
Vercel Dashboard → Deployments → Redeploy
```

### 数据库连接失败？

```bash
# 检查 DATABASE_URL
确认格式：postgresql://user:pass@host/db?sslmode=require
确认包含 ?sslmode=require

# 测试连接
Render Shell:
python -c "from app.database import engine; print(engine.url)"
```

### AI API 不工作？

```bash
# 检查 DeepSeek 余额
访问 https://platform.deepseek.com/
查看账户余额

# 检查 API Key
确认以 sk- 开头
确认没有多余的空格

# 检查配置
OPENAI_BASE_URL=https://api.deepseek.com/v1
OPENAI_MODEL=deepseek-chat
```

## 💡 优化建议

### 防止后端休眠

1. 注册 https://uptimerobot.com/
2. Add New Monitor:
   - Type: HTTP(s)
   - URL: 你的后端 URL
   - Interval: 5 minutes

### 查看实时日志

```bash
# Render 日志
Render Dashboard → Logs → 实时查看

# Vercel 日志
Vercel Dashboard → Deployments → 点击部署 → View Function Logs
```

### 更新代码

```bash
# 后端自动部署
推送到 GitHub → Render 自动重新部署

# 前端自动部署
推送到 GitHub → Vercel 自动重新部署
```
